# Pome Jarvis MVP 기능 명세

이 문서는 수정된 MVP 계획안을 코드 작업 기준으로 쪼갠 기능 명세다. 4주 MVP는 외부 서비스 완전 연동이 아니라 핵심 UX, 상태 저장, 승인 흐름, 실행 로그 검증을 목표로 한다.

## 1. 오늘의 운영판

| 항목 | 내용 |
|------|------|
| 입력 | 샘플 일정, 샘플 메일, 작업 목록, 승인 대기 목록 |
| 출력 | 중요 메일 수, 열린 할 일 수, 승인 대기 수, 오늘 일정, 추천 액션 |
| 완료 조건 | 사용자가 앱 첫 화면에서 오늘 확인할 핵심 항목을 1분 안에 파악할 수 있다 |

## 2. 이메일 요약

| 항목 | 내용 |
|------|------|
| 입력 | 메일 제목, 발신자, 본문, 수신 시간 |
| 출력 | `urgent`, `reply_needed`, `reference` 라벨과 요약 |
| 완료 조건 | 샘플 메일이 긴급, 답장 필요, 참고로 명확히 구분된다 |

## 3. 답장 초안

| 항목 | 내용 |
|------|------|
| 입력 | 선택된 메일 |
| 출력 | 승인 대기함에 `email_send` 항목 생성 |
| 완료 조건 | 실제 발송 없이 초안 제안과 승인 로그가 생성된다 |

## 4. 회의록 액션 추출

| 항목 | 내용 |
|------|------|
| 입력 | 회의록 텍스트 |
| 출력 | 작업 목록에 `meeting` 출처의 할 일 추가 |
| 완료 조건 | 담당자, 기한, 작업 제목이 사용 가능한 형태로 생성된다 |

## 5. 승인 대기함

| 항목 | 내용 |
|------|------|
| 입력 | 실행 후보 항목 |
| 출력 | `approved` 또는 `rejected` 상태와 실행 로그 |
| 완료 조건 | 발송/변경 같은 위험 작업은 바로 실행되지 않고 승인 기록만 남는다 |

## 6. 실행 로그

| 항목 | 내용 |
|------|------|
| 입력 | 브리핑 생성, 초안 생성, 액션 추출, 승인/거절 이벤트 |
| 출력 | 시간, 액션명, 상세 내용, 상태 |
| 완료 조건 | 사용자가 비서가 한 일을 추적할 수 있다 |

## 7. 저장 정책

MVP는 서버 DB 없이 `localStorage`에 데모 상태를 저장한다. 저장 대상은 작업 목록, 승인 대기함, 실행 로그, 회의록 입력값, 선택된 메일이다. 이후 API/DB가 추가되면 서비스 계층만 교체한다.
